# ADR Template for DiriCode

> Use this template for all Architecture Decision Records.
> Language: English.
> Date format: YYYY-MM-DD.

---

```markdown
## ADR-XXX — [Short Decision Title]

| Field       | Value                        |
|-------------|------------------------------|
| Status      | Accepted / Superseded / Draft |
| Date        | YYYY-MM-DD                   |
| Scope       | MVP / v2 / v3                |
| References  | [analysis docs, prior ADRs]  |

### Context

Why is this decision needed? What problem or constraint drives it?

### Decision

What was decided. Be specific. Include configuration values, format choices, tool names, etc.

### Consequences

- **Positive:** What improves because of this decision.
- **Negative / Trade-offs:** What becomes harder or is deferred.
- **Migration notes:** If this supersedes a prior decision, what changes.

### Details (optional)

Tables, schemas, examples, or other specifics that belong with this ADR.
```
