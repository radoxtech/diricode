Ewolucja Architektury: DiriCode

Wizualne porównanie oryginalnego OpenCode, modyfikacji Oh-My-OpenCode oraz nowej, zoptymalizowanej architektury DiriCode.

1. OpenCode (Oryginał)

Monolityczna architektura z ciężkimi zależnościami. Proste agenty, ale ogromny narzut na system (28 serwerów LSP, 22 wbudowanych providerów).

Ogromne node_modules] Providers --> LLM[(LLM APIs)]
  • Stan: Baza SQLite (trudna w edycji ręcznej).
  • Agenty: Proste, zahardcodowane w kodzie.
  • Routing: Brak zaawansowanego routingu, bezpośrednie strzały do providerów.
  • Bezpieczeństwo: Brak wbudowanego approval workflow, agent może zepsuć repozytorium (`git add .`).

2. Oh-My-OpenCode (Fork)

Potężny, ale "przeinżynierowany" system. Wprowadza świetne narzędzia (Hashline, AST), ale tonie w 44 hookach i gigantycznym kontekście agenta Sisyphus.

Ogromny narzut tokenów] Hooks --> Sisyphus[Sisyphus Agent
Do-it-all, Monolityczny Kontekst] Sisyphus --> SubAgents[Sub-Agenci
Max 2 poziomy] Sisyphus --> Tools[Zaawansowane Narzędzia] Tools --> Hashline[Hashline Edit] Tools --> AST[AST-grep] Sisyphus --> Fallback[Model Fallback Chain
7 modeli w łańcuchu] Fallback --> LLM[(LLM APIs)]
  • Stan: SQLite + hackowanie plików TODO przez agenta.
  • Agenty: Sisyphus robi wszystko, ma dostęp do wszystkich hooków, co drastycznie zwiększa koszty tokenów.
  • Routing: Wbudowany łańcuch fallbacków, ale brak równoległych zapytań (Race Mode).
  • Złożoność: Dynamiczny prompt builder jest bardzo trudny w utrzymaniu.

3. DiriCode (Twoja Architektura)

Lekka, bezpieczna i modularna architektura. Agent główny tylko dyryguje (Dispatcher-First), stan żyje w plikach Markdown, a własny router TS zarządza failoverem i wyścigami modeli.

  • Stan: Pliki Markdown (`todo.md`) zamiast SQLite. Pełna kontrola w Git.
  • Agenty: Dispatcher-First (tani, szybki). Sub-agenci są hybrydowi (logika w TS, instrukcje w edytowalnych plikach `.md`).
  • Routing: Własny TS Router z obsługą Race Mode (równoległe zapytania) i Order-based Failover.
  • Bezpieczeństwo: Wbudowana redakcja sekretów, Tree-sitter do Basha, Git Safety Rails, Smart Approval Workflow.
